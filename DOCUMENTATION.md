# Часть I — Документация разработчика

## 1. Архитектура системы

Приложение построено по архитектуре клиент-сервер:

```
[ React SPA ]  ←──── HTTP/JSON ────→  [ FastAPI ]  ←──→  [ PostgreSQL ]
                                                    ←──→  [ Redis Cache ]
                                                    ←──→  [ Gemini AI API ]
```

- **Frontend** — одностраничное приложение (SPA) на React. Взаимодействует с бэкендом через REST API. Токены JWT хранятся в `localStorage` и автоматически подставляются в заголовки через axios-интерцепторы.
- **Backend** — асинхронный REST API на FastAPI. Реализует бизнес-логику, работу с БД через SQLAlchemy 2.0 (async), кэширование через Redis.
- **PostgreSQL** — основное хранилище данных (пользователи, вопросы, попытки, статистика).
- **Redis** — кэш для списков вопросов по темам (TTL: 600 секунд).

---

## 2. Описание API

Базовый URL всех эндпоинтов: `/api/v1`

Аутентификация: Bearer JWT токен в заголовке `Authorization: Bearer <token>` (кроме `/auth/register` и `/auth/login`).

---

### 2.1 Аутентификация — `/auth`

| Метод | URL | Описание | Доступ |
|---|---|---|---|
| `POST` | `/auth/register` | Регистрация нового пользователя. Тело: `{email, username, password, language_pref?, target_ort_score?}` | Публичный |
| `POST` | `/auth/login` | Вход. Форма OAuth2 (`username` = email, `password`). Возвращает `access_token` и `refresh_token`. | Публичный |
| `POST` | `/auth/refresh` | Обновление токенов. Тело: `{refresh_token}` | Публичный |
| `GET` | `/auth/me` | Получить профиль текущего пользователя | Авторизован |
| `PATCH` | `/auth/me` | Обновить профиль (язык, целевой балл и др.) | Авторизован |

### 2.3 Вопросы — `/questions`

|---|---|---|---|
| `GET` | `/questions/exam?type=&limit=` | Случайная выборка верифицированных вопросов для симуляции ОРТ. Параметры: `type` (`comparison` / `standard`), `limit` (1–50) | Авторизован |
| `GET` | `/questions/by-topic/{topic_id}?difficulty=&limit=` | Вопросы по теме без правильных ответов (режим теста). Кэшируется. | Авторизован |
| `GET` | `/questions/{question_id}` | Полные данные вопроса (с правильным ответом) | Авторизован |
| `POST` | `/questions/` | Создать вопрос вручную | Администратор |
| `PATCH` | `/questions/{question_id}` | Обновить вопрос | Администратор |

### 2.4 Попытки и прогресс — `/attempts`

| Метод | URL | Описание | Доступ |
|---|---|---|---|
| `POST` | `/attempts/submit` | Отправить ответ. Тело: `{question_id, selected_answer, time_spent_seconds}`. Запускает логику XP, стрика и архива ошибок. | Авторизован |
| `GET` | `/attempts/errors?mastered=` | Архив ошибок пользователя с данными вопросов. Фильтр: `mastered=true/false` | Авторизован |
| `PATCH` | `/attempts/errors/{error_id}/retry` | Повторная попытка задачи из архива. Логика: 3 верных ответа подряд → `mastered=true`. | Авторизован |
| `GET` | `/attempts/history?limit=` | История последних попыток (до 50) | Авторизован |
| `GET` | `/attempts/stats` | Общая статистика: точность, XP, прогноз балла ОРТ, стрик | Авторизован |
| `GET` | `/attempts/streak` | Текущий дневной стрик и XP | Авторизован |

## 4. Модуль безопасности

- **JWT**: токены создаются через `python-jose`. Access-токен действует 30 минут, refresh-токен — 7 дней.
- **Хэширование паролей**: bcrypt через `passlib`.
- **Автоматическое обновление токенов**: axios-интерцептор на фронтенде перехватывает ответ `401 Unauthorized` и автоматически выполняет refresh-запрос.
- **Кэширование**: Redis-клиент использует TTL 600 секунд для списков вопросов по темам; кэш инвалидируется при добавлении или обновлении вопроса.

# Часть II — Пользовательская документация

## Руководство пользователя — ORT Math Platform

### Шаг 1: Регистрация и вход

1. Откройте приложение в браузере.
2. На странице входа нажмите **«Нет аккаунта? Зарегистрироваться»**.
3. Введите ваш email, придумайте имя пользователя и пароль (не менее 6 символов).
4. Нажмите **«Создать аккаунт»** — система автоматически выполнит вход.

> Для повторного входа используйте email и пароль на той же странице.

---

### Шаг 2: Изучение Dashboard

После входа вы попадёте на главную страницу (**Dashboard**), где отображается:

- **Прогноз балла ОРТ** — расчётный балл на основе ваших ответов (шкала 110–245).
- **Точность** — процент верных ответов за всё время.
- **Серия дней (Streak)** и накопленный **XP** — мотивационные показатели.
- **Темы** — карточки разделов математики (Арифметика, Алгебра, Геометрия, Анализ данных). Нажмите на любую, чтобы начать тренировку по этой теме.

---

### Шаг 3: Тренировка и режимы практики

Перейдите в раздел **Practice** через боковое меню. Доступны три режима:

**Тематическая тренировка** — нажмите на одну из четырёх тем. Отвечайте на вопросы в своём темпе; после каждого ответа вы увидите объяснение с пошаговым решением.

**Спринт** — 20 задач на время (20–30 минут). По завершении система показывает результат и разбор допущенных ошибок.

**Симуляция ОРТ** — нажмите **«Mock Exam»** в боковом меню. Запустится полная симуляция: 60 вопросов в двух частях (сравнение величин + основная математика) с таймером. Экзамен проходит в полноэкранном режиме.

---

### Шаг 4: Архив ошибок и система мастерства

Все задачи, в которых вы ошиблись, автоматически попадают в раздел **Error Archive** (боковое меню).

- Каждая задача имеет прогресс **0/3** — нужно ответить верно три раза подряд.
- При каждой ошибке прогресс сбрасывается до нуля.
- Как только задача достигает **3/3** — она помечается как освоенная и исчезает из активного списка.
- За каждый верный ответ в архиве начисляется **+20 XP**.

> Регулярная работа с архивом ошибок — наиболее эффективный способ повысить прогноз балла ОРТ.
