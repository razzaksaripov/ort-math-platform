# Отчёт по тестированию — ORT Math Platform

**Дата тестирования:** Апрель 2026  
**Версия приложения:** 1.0.0  
**Среда:** Development (localhost)

---

## Методология

Тестирование проводилось вручную через пользовательский интерфейс и автоматически через Swagger UI (`/docs`). Проверялись ключевые сценарии: аутентификация, работа с данными, бизнес-логика платформы и обработка граничных случаев.

---

## Таблица тест-кейсов

| № | Название функции | Описание действия | Ожидаемый результат | Статус |
|---|---|---|---|---|
| TC-01 | Регистрация нового пользователя | Отправить POST `/api/v1/auth/register` с валидными данными: email, username, password | Сервер возвращает `201 Created` с объектом пользователя; в таблице `users` создаётся новая запись; автоматически создаётся запись `daily_streaks` | ✅ Passed |
| TC-02 | Аутентификация (логин) | Отправить POST `/api/v1/auth/login` с корректными email и паролем через форму OAuth2 | Сервер возвращает `200 OK` с `access_token` и `refresh_token`; токены сохраняются в `localStorage` через Zustand | ✅ Passed |
| TC-03 | Получение данных из базы данных | Авторизованный пользователь открывает страницу `/dashboard` | Frontend выполняет GET `/api/v1/topics/` и `/api/v1/attempts/stats`; данные корректно отображаются в карточках статистики; ответ кэшируется в Redis | ✅ Passed |
| TC-04 | Отправка ответа и начисление XP | Пользователь выбирает верный ответ на вопрос и нажимает «Проверить ответ» | POST `/api/v1/attempts/submit` возвращает `is_correct: true`, начисляет XP; запись сохраняется в `user_attempts`; обновляется `daily_streaks.xp_points` | ✅ Passed |
| TC-05 | Система архива ошибок и мастерства | Пользователь трижды подряд верно отвечает на задачу в разделе «Архив ошибок» | После третьего верного ответа (`success_count = 3`) поле `mastered` становится `true`; задача анимированно исчезает из списка; пользователь получает +20 XP | ✅ Passed |
| TC-06 | Обработка ошибки аутентификации | Отправить запрос к защищённому эндпоинту без токена или с истёкшим токеном | Сервер возвращает `401 Unauthorized`; axios-интерцептор автоматически выполняет refresh-запрос; при невалидном refresh-токене происходит редирект на `/login` | ✅ Passed |
| TC-07 | ИИ-парсинг изображения с задачами | Администратор загружает фотографию страницы с задачами ОРТ через `/admin` | POST `/api/v1/admin/parse-image` передаёт изображение в Gemini API; возвращается структурированный JSON-массив с полями `content_latex`, `options`, `correct_answer`, `explanation`; данные отображаются для проверки перед сохранением | ✅ Passed |

## Граничные случаи

| Сценарий | Действие | Результат |
|---|---|---|
| Регистрация с занятым email | POST `/auth/register` с уже существующим email | `409 Conflict` — `AlreadyExistsException` |
| Запрос вопросов без данных в БД | GET `/questions/exam` при пустой базе | Возвращается пустой массив `[]`, без ошибки 500 |
| Повторный retry освоенной задачи | PATCH `/attempts/errors/{id}/retry` при `mastered=true` | `400 Bad Request` с сообщением «Задача уже освоена» |

## Вывод

Все 7 основных тест-кейсов успешно пройдены. Ключевые функции платформы — аутентификация, работа с базой данных, геймификация и ИИ-парсинг — функционируют корректно в соответствии с требованиями.
